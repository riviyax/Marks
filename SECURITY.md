# Security Policy

## Supported Versions

Only the latest version on `main` is actively supported with security updates. Older commits, tags, or forks are not maintained — please update to the latest `main` before reporting an issue.

| Version          | Supported          |
| ----------------- | ------------------ |
| Latest (`main`)   | :white_check_mark: |
| Anything older    | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability in Marks System (bot, client, or server), please report it privately rather than opening a public issue.

You can report through either of the following:

- **GitHub Security Advisories** — open a private report via the [Security tab](../../security/advisories/new) on this repository.
- **Email** — send details to **support@riviyax.site**.

When reporting, please include:

- A description of the vulnerability and its potential impact
- Steps to reproduce, or a proof of concept if available
- Which part of the project is affected (`bot`, `client`, or `server`)

### What to expect

- You'll receive an acknowledgment within **48–72 hours** of your report.
- We'll investigate and aim to provide a status update within **7 days**.
- If the vulnerability is confirmed, a fix will be prioritized and released on `main`, and you'll be credited (unless you prefer to remain anonymous).
- If the report is declined (e.g. not reproducible, out of scope, or by design), we'll explain the reasoning.

Please do not publicly disclose the issue until it has been addressed.
